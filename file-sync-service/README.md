# 🔄 Сервис синхронизации файлов с Яндекс Диском

Автоматический сервис для синхронизации файлов между локальным компьютером и облачным хранилищем Яндекс Диск. Программа отслеживает изменения в файлах и папках, автоматически загружая, обновляя и удаляя данные в облаке.

##  Возможности

- 🔄 **Автоматическая синхронизация** — программа работает в фоновом режиме с настраиваемым интервалом.
-  **Поддержка вложенных папок** — рекурсивная синхронизация структуры любой глубины.
- 📤 **Загрузка новых файлов** — автоматическая загрузка при появлении новых файлов.
- 🔄 **Обновление изменённых файлов** — сравнение по времени изменения (mtime).
- 🗑️ **Удаление файлов** — синхронное удаление из облака при удалении локально.
- 📂 **Создание структуры папок** — автоматическое создание вложенных папок.
- 📝 **Подробное логирование** — все действия записываются в лог-файл и выводятся в консоль.
- 🛡️ **Глубокая обработка ошибок** — использование кастомных исключений и устойчивая работа при проблемах с сетью.
- 🔒 **Безопасность** — токены и настройки хранятся в `.env` файле.

## 🛠️ Технологии

- **Python 3.8+**
- **Requests** — HTTP-запросы к API Яндекс Диска.
- **python-dotenv** — управление конфигурацией через `.env` файлы.
- **loguru** — продвинутое логирование (замена стандартных `print`).
- **Яндекс Диск API** — облачное хранилище.

## 📋 Требования

- Python 3.8 или выше.
- Аккаунт Яндекс.
- Токен доступа к Яндекс Диску (OAuth).

## 🏗️ Архитектура проекта

Проект разделён на логические слои для соблюдения принципа единой ответственности (SRP):

- **`main.py`** — Точка входа. Содержит только вызов функции запуска.
- **`app.py`** — Слой приложения. Отвечает за инициализацию зависимостей, валидацию окружения и запуск основного цикла.
- **`sync_service.py`** — Слой бизнес-логики. Содержит алгоритм сравнения и синхронизации файлов.
- **`file_manager.py`** и **`yandex_disk.py`** — Инфраструктурные слои. Инкапсулируют работу с локальной файловой системой и внешним API.
- **`config.py`** — Отвечает за безопасную загрузку и валидацию настроек.
- **`ui.py`** — Отвечает за форматирование и вывод сообщений (используется `loguru` вместо `print`).
- **`exceptions.py`** — Иерархия кастомных исключений для точного отлова и обработки ошибок.

##  Как это работает

1. **Инициализация:** `main.py` вызывает `run_service()` из `app.py`. Загружается и валидируется конфигурация из `.env`.
2. **Проверка окружения:** Проверяется существование локальной папки. Инициализируется клиент Яндекс Диска, при необходимости создаётся корневая папка в облаке.
3. **Сбор информации:** Рекурсивно сканируется локальная папка (`file_manager.py`) и облако (`yandex_disk.py`).
4. **Синхронизация (`sync_service.py`):**
   - Создаёт отсутствующие папки в облаке.
   - Загружает новые файлы.
   - Обновляет изменённые файлы (по времени модификации).
   - Удаляет файлы и папки, которых нет локально.
5. **Цикл:** Повторяет процесс каждые N секунд. При возникновении сетевых или API ошибок сервис логирует их и продолжает работу в следующем цикле.

## 🐛 Обработка ошибок

В проекте реализована многоуровневая обработка исключений:
- **`ConfigError`** — при некорректных настройках в `.env`.
- **`LocalFolderError`** — при проблемах с правами доступа или отсутствием локальной папки.
- **`NetworkError`** — при обрыве соединения или таймаутах API.
- **`CloudAPIError`** — при ошибках со стороны Яндекс Диска (401, 403, 500).
- **`FileOperationError` / `FolderOperationError`** — при ошибках чтения/записи/удаления.

Сервис не падает при разовых сетевых сбоях, а корректно логирует ошибку и переходит к следующему циклу синхронизации.
