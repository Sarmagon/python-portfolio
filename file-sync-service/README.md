# 🔄 Сервис синхронизации файлов с Яндекс Диском

![Python](https://img.shields.io/badge/Python-3.8+-blue?logo=python)
![License](https://img.shields.io/badge/license-MIT-green)
![Status](https://img.shields.io/badge/status-active-success)

Автоматический сервис для синхронизации файлов между локальным компьютером и облачным хранилищем Яндекс Диск. Программа отслеживает изменения в файлах и папках, автоматически загружая, обновляя и удаляя данные в облаке.

## ✨ Возможности

- 🔄 **Автоматическая синхронизация** - программа работает в фоновом режиме с настраиваемым интервалом
- 📁 **Поддержка вложенных папок** - рекурсивная синхронизация структуры любой глубины
- 📤 **Загрузка новых файлов** - автоматическая загрузка при появлении новых файлов
- 🔄 **Обновление изменённых файлов** - сравнение по времени изменения
- 🗑️ **Удаление файлов** - синхронное удаление из облака при удалении локально
- 📂 **Создание структуры папок** - автоматическое создание вложенных папок
- 📝 **Подробное логирование** - все действия записываются в лог-файл
- 🎨 **Красивый вывод** - информативный вывод с эмодзи в терминал
- 🛡️ **Обработка ошибок** - устойчивая работа при проблемах с сетью или файлами
- 🔒 **Безопасность** - токены и настройки хранятся в `.env` файле

## 🛠️ Технологии

- **Python 3.8+**
- **Requests** - HTTP-запросы к API Яндекс Диска
- **python-dotenv** - управление конфигурацией через `.env` файлы
- **loguru** - продвинутое логирование
- **Яндекс Диск API** - облачное хранилище

## 📋 Требования

- Python 3.8 или выше
- Аккаунт Яндекс
- Токен доступа к Яндекс Диску

## 🚀 Установка

1. **Клонируйте репозиторий:**
```bash
git clone https://github.com/ваш-username/file-sync-service.git
cd file-sync-service
```

2. **Установите зависимости:**
```bash
pip install requests python-dotenv loguru
```

3. **Получите токен Яндекс Диска:**
   - Перейдите на https://yandex.ru/dev/disk/poligon/
   - Войдите в свой аккаунт Яндекс
   - Сгенерируйте токен OAuth

4. **Настройте файл `.env`:**
```env
LOCAL_FOLDER=C:/путь/к/вашей/папке
YANDEX_FOLDER=имя_папки_на_диске
YANDEX_TOKEN=ваш_токен_здесь
SYNC_PERIOD=10
LOG_FILE=sync.log
```

**Параметры:**
- `LOCAL_FOLDER` - путь к локальной папке для синхронизации
- `YANDEX_FOLDER` - имя папки в облаке (создастся автоматически)
- `YANDEX_TOKEN` - ваш токен доступа к Яндекс Диску
- `SYNC_PERIOD` - интервал синхронизации в секундах
- `LOG_FILE` - имя файла для логов

## 🔧 Как это работает

1. **Инициализация:** Программа читает настройки из `.env`, создаёт логгер и экземпляр класса `YandexDisk`
2. **Проверка облака:** Проверяет существование папки на Яндекс Диске, создаёт её при необходимости
3. **Сбор информации:** Рекурсивно сканирует локальную папку и облако, собирая информацию о файлах и папках
4. **Синхронизация:**
   - Создаёт отсутствующие папки в облаке
   - Загружает новые файлы
   - Обновляет изменённые файлы (по времени модификации)
   - Удаляет файлы и папки, которых нет локально
5. **Цикл:** Повторяет процесс каждые N секунд (настраивается в `.env`)

## 🎯 Примеры использования

### Синхронизация документов
Синхронизируйте рабочую папку с документами для резервного копирования:
```env
LOCAL_FOLDER=C:/Users/username/Documents/Work
YANDEX_FOLDER=work_backup
SYNC_PERIOD=300  # каждые 5 минут
```

### Синхронизация проектов
Создайте бэкап своих проектов:
```env
LOCAL_FOLDER=C:/Projects
YANDEX_FOLDER=projects_backup
SYNC_PERIOD=600  # каждые 10 минут
```

## 📝 Логирование

Все действия записываются в файл `sync.log`:
- Создание папок и загрузка файлов
- Обновление изменённых файлов
- Удаление файлов и папок
- Ошибки и исключения

Формат лога:
```
synchroniser 2026-07-11 16:48:40,425 INFO Файл test1.txt успешно загружен.
synchroniser 2026-07-11 16:48:42,714 INFO Папка Documents создана на облаке.
```

## 🐛 Обработка ошибок

Программа устойчива к ошибкам:
- **Сетевые проблемы** - программа продолжит работу после восстановления соединения
- **Отсутствие файлов** - пропустит недоступные файлы с записью в лог
- **Ошибки API** - обработает и залоггирует ошибки Яндекс Диска
- **Неверные настройки** - выведет понятное сообщение об ошибке

## 🔐 Безопасность

- Токен хранится только в `.env` файле
- `.env` добавлен в `.gitignore` и не попадает в Git
- Токен не выводится в логи или консоль
