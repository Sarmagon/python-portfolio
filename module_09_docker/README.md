### Задача 2. Dockerfile
Написан оптимизированный Dockerfile для приложения «Обратный отсчёт до Нового года»:

```dockerfile
FROM python:3.9-slim

# Рабочая директория в контейнере
WORKDIR /app

# Установка зависимостей (отдельный слой для кеширования)
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Копирование кода приложения
COPY . .

# Открытие порта
EXPOSE 5000

# Запуск приложения (массивная форма CMD)
CMD ["python", "app.py"]
```

**Принципы оптимизации:**
- ✅ Операции с часто изменяемыми файлами (`COPY . .`) расположены ближе к концу
- ✅ Несколько команд `RUN` объединены в одну для уменьшения количества слоёв
- ✅ Используется массивная форма `CMD` вместо строковой (корректная обработка `Ctrl+C`)
- ✅ Базовый образ `slim` для минимизации размера

## 🚀 Запуск проекта

### Локально (без Docker)

```bash
# Установка зависимостей
pip install -r requirements.txt

# Запуск приложения
python app.py
```

Приложение будет доступно по адресу: http://localhost:5000

### В Docker

```bash
# Сборка образа
docker build -t new-year-app .

# Запуск контейнера
docker run -p 5000:5000 new-year-app
```

### На удалённом сервере

```bash
# Загрузка образа из Docker Hub
docker pull sarmagon/new-year-app

# Запуск с автозапуском при перезагрузке
docker run -d -p 5000:5000 --restart=always --name new-year-app sarmagon/new-year-app
```

## 📊 Мониторинг

```bash
# Просмотр запущенных контейнеров
docker ps

# Вход в контейнер для мониторинга
docker exec -it new-year-app bash

# Установка и запуск htop
apt-get update && apt-get install -y htop
htop

# Определение пути установки
which htop
```
